# Сниппет: подстановка почт по источнику трафика

Один скрипт на **site.ru**. Почты берутся из настроек проекта в CRM — ничего не генерируется.

## Подключение

```html
<script
  src="https://crm.mv-deploy.ru/embed/wbooster.js"
  data-token="SITE_UUID:secret"
  async
></script>
```

Токен — админка → **Проект** → «Токен интеграции».

## Какую почту показывает посетителю

| Источник визита | Поле в CRM | Когда срабатывает |
|-----------------|------------|-------------------|
| **Реклама** | Почта (реклама) | `utm_medium=cpc/ppc/paid…`, метки `yclid`, `gclid`, `fbclid`, рекламные UTM |
| **Поиск (ПИ)** | Почта (SEO / поиск) | Переход с Google, Yandex и др. без платных меток |
| **Прямой заход** | Почта (прямые заходы) | Нет referrer, закладка, ввод URL вручную |

Скрипт запоминает источник **на время сессии** (sessionStorage): при переходе по страницам site.ru почта не меняется.

## Что подменяется на странице

- все ссылки `mailto:…`
- элементы с `data-wbooster-email` (если нужно явно пометить блок)

## Настройка в CRM

На карточке проекта три поля:

1. **Почта (реклама)** — для платного трафика  
2. **Почта (SEO / поиск)** — для органики из поисковиков  
3. **Почта (прямые заходы)** — для прямых визитов  

На каждый ящик — **пересылка** на служебный IMAP CRM ([integraciya-inbound-email.md](../integraciya-inbound-email.md)).

## Проверка

1. Откройте site.ru?utm_medium=cpc — в `mailto:` должна быть рекламная почта.  
2. Перейдите с поиска Yandex (или эмулируйте referrer) — SEO-почта.  
3. Откройте site.ru напрямую в новой вкладке — почта «прямые заходы».
